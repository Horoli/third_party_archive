# 다국어 및 정규식 로직 (I18N.md)

이 문서는 프로젝트의 다국어(i18n) 시스템, 보관함 검색용 정규식(Regex) 생성 규칙, 그리고 견고한 날짜 파싱 로직을 설명합니다.

## 1. 전역 다국어 시스템
- **컨트롤러 중심 제어**: `GetDashboard` 컨트롤러의 `isKorean` 상태를 앱 전체가 공유합니다.
- **실시간 바인딩**: `Obx` 위젯을 사용하여 언어 전환 즉시 UI가 업데이트됩니다.
- **데이터 소스**: `lib/preset/i18n.dart`의 `SCARAB` 맵을 통해 명칭, 분류, 전용 정규식을 관리합니다.

## 2. 지능형 정규식 (Regex) 생성 규칙
사용자가 시세 버튼을 클릭하면 클립보드에 복사되는 정규식은 다음 규칙을 따릅니다.

### 영문 모드 (English)
- **전체 접두어**: 최종 생성된 문자열 맨 앞에 `"scarab "`을 한 번만 추가하여 갑충석 탭 필터링을 최적화합니다.
    - 예: `scarab dlines|cont|^scarab of w`
- **선별적 정교화 (`^`)**: 이름이 겹칠 위험이 큰 항목(예: `Scarab of Wisps`)은 `^scarab of ` 패턴을 사용하여 `Ritual Scarab` 등과 확실히 구분합니다.
- **우선순위**:
    1. **전용 키워드**: `i18n.dart`의 `regexEn` 필드값 우선 사용. (예: `dlines\$`, `wak`)
    2. **Fallback 알고리즘**: `"of"` 포함 시 `[첫글자].of.[끝단어앞3글자]`, 미포함 시 `[첫단어앞3글자]`.

### 국문 모드 (Korean)
- **접두어 없음**: 국문 검색 시스템에 맞춰 별도의 접두어 없이 생성합니다.
- **우선순위**:
    1. **전용 키워드**: `i18n.dart`의 `regexKr` 필드값 우선 사용. (예: `맹의.뿔`)
    2. **Fallback 알고리즘**: `[첫단어 끝2글자].[두번째단어 첫1글자]` (예: `고무의 역병` -> `무의.역`).

## 3. 견고한 날짜 파싱 (Robust Date Parsing)
다양한 로케일 형식(특히 한국어 오전/오후 포함)의 날짜 문자열을 표준화합니다.

- **로케일 대응**: 정규표현식을 사용하여 `2026. 3. 18. 오전 12:16:18`와 같은 문자열에서 년, 월, 일, 시, 분을 추출합니다.
- **24시간제 변환**: "오전/오후" 문구를 해석하여 24시간 형식으로 자동 계산합니다.
- **표준 출력**: 모든 날짜는 `yyyy-MM-dd HH:mm` 형식으로 표시됩니다.

## 4. 개발 주의사항
- **이스케이프**: Dart 문자열 내 정규식 기호(`$`) 사용 시 반드시 `\$`로 이스케이프 처리해야 합니다.
- **중복 방지**: 영문 `regexEn` 정의 시 이미 `^`나 `scarab`으로 시작하는 경우 복사 로직에서 중복 처리를 하지 않도록 설계되었습니다.
