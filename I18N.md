# 다국어 및 정규식 로직 (I18N.md)

이 문서는 프로젝트의 다국어(i18n) 시스템, 보관함 검색용 정규식(Regex) 생성 규칙, 그리고 견고한 날짜 파싱 로직을 설명합니다.

## 1. 전역 다국어 시스템
- **컨트롤러 중심 제어**: `GetDashboard` 컨트롤러의 `isKorean` 상태를 앱 전체가 공유합니다.
- **실시간 바인딩**: `Obx` 위젯을 사용하여 언어 전환 즉시 UI가 업데이트됩니다.
- **데이터 소스**: `lib/preset/i18n.dart`의 `SCARAB` 맵을 통해 명칭, 분류, 전용 정규식을 관리합니다.

## 2. 지능형 정규식 (Regex) 생성 규칙
사용자가 시세 버튼을 클릭하면 클립보드에 복사되는 정규식은 다음 규칙을 엄격히 따릅니다.

### 공통 규칙
- **큰따옴표 감싸기**: 띄어쓰기 인식 오류를 방지하기 위해 정규식 본문을 반드시 큰따옴표(`"`)로 묶습니다.
- **제외 필터(`!`) 접두사 (2026-03-21)**: 특정 항목들을 제외하고 검색하고 싶을 때, 큰따옴표 바로 뒤에 느낌표(`!`)를 붙여 정규식을 생성합니다. 
    - 사용 예: **1c↓ (정리)** 버튼 클릭 시, 가치 있는(1c 이상) 모든 갑충석을 제외하여 잡 갑충석만 남깁니다.
    - 포맷: `갑충 "!regex|regex"` 또는 `scarab "!regex|regex"`
- **마침표(`.`) 치환**: 단어 사이의 공백은 정규식 와일드카드인 마침표(`.`)로 치환하여 글자 수를 절약하고 검색 정확도를 높입니다.
- **최단 고유 문자열**: `i18n.dart`에 정의된 `regexKr`, `regexEn`은 다른 갑충석과 충돌하지 않는 가장 짧은 문자열을 지향합니다.

### 영문 모드 (English)
- **접두어 추가**: 최종 결과물 맨 앞에 `scarab `을 추가합니다.
    - 포맷: `scarab "regex|regex"` 또는 `scarab "!regex|regex"` (제외 시)
    - 예: `scarab "!dlines|cont|^scarab.of.w"`
- **기본 갑충석 (Base)**: 수식어가 없는 기본 갑충석은 끝 앵커(`$`)를 활용합니다. (예: `abys.*b$`)
- **와일드카드 압축**: `[첫단어 끝3자].*.[막단어 시작3자]` 패턴을 활용합니다. (예: `yss.*mul`)

### 국문 모드 (Korean)
- **접두어 추가**: 최종 결과물 맨 앞에 `갑충 `을 추가합니다.
    - 포맷: `갑충 "정규식|정규식"` 또는 `갑충 "!정규식|정규식"` (제외 시)
    - 예: `갑충 "!회랑|냄의.점|^심연"`
- **기본 갑충석 (Base)**: 수식어가 앞에 붙는 국문 특성에 맞춰 시작 앵커(`^`)를 활용합니다. (예: `^역병`)
- **Fallback 알고리즘**: `[앞단어 끝2자].[뒷단어 시작1자]` (예: `고무의 역병` -> `무의.역`)

## 3. 견고한 날짜 파싱 (Robust Date Parsing)
다양한 로케일 형식(특히 한국어 오전/오후 포함)의 날짜 문자열을 표준화합니다.

- **로케일 대응**: 정규표현식을 사용하여 `2026. 3. 18. 오전 12:16:18`와 같은 문자열에서 년, 월, 일, 시, 분을 추출합니다.
- **24시간제 변환**: "오전/오후" 문구를 해석하여 24시간 형식으로 자동 계산합니다.
- **표준 출력**: 모든 날짜는 `yyyy-MM-dd HH:mm` 형식으로 표시됩니다.

## 4. 개발 주의사항
- **이스케이프**: Dart 문자열 내 정규식 기호(`$`) 사용 시 반드시 `\$`로 이스케이프 처리해야 합니다.
- **데이터 정합성**: `lib/preset/i18n.dart`를 수동 업데이트할 때는 `SCARAB_LOCATION` 맵의 순서와 일치하는지 확인해야 합니다.

## 5. GetX 및 성능 최적화
- **Lifted Obx Pattern**: 수백 개의 그리드 셀마다 리스너를 두지 않고 상위 수준에서 한 번만 언어 상태를 관찰하여 렌더링 성능을 극대화했습니다.
- **O(N+M) 연산**: 아이템 필터링 및 선택 상태 확인 시 `Set`을 활용하여 선형 시간 복잡도를 보장합니다.
