# UI 작업 내용 (UI.md)

이 문서는 프로젝트의 사용자 인터페이스(UI)와 관련된 최신 변경 사항을 기록합니다.

## 1. 전역 동기화 및 언어 제어 (FAB 영역)
- **통합 정보 인터페이스**: 화면 우측 하단에 `[동기화 시간 위젯] [언어 전환 버튼]`을 나란히 배치했습니다.
- **WidgetNinjaSyncInfo**: poe.ninja와의 최신 데이터 동기화 시간을 전역적으로 표시합니다.
    - **디자인**: 반투명 검은색 배경의 깔끔한 칩 형태.
    - **상호작용**: 호버 시 "데이터는 poe.ninja와 연동됩니다" 툴팁 노출.
- **Floating Language Button**: 현재 언어(`KO`/`EN`)를 즉시 전환하며, 앱 내의 모든 반응형 UI가 즉각 대응합니다.

## 2. 갑충석 시세표 (Scarab Price Table)
- **개별 선택 모드 (Custom Selection)**: 갑충석 타일을 클릭하여 원하는 항목만 선택할 수 있습니다. 
    - **시각적 피드백**: 선택된 아이템은 **흰색 굵은 테두리**, **체크 아이콘**, **불투명도 100%**로 강조됩니다.
    - **커스텀 복사**: 선택된 항목이 있을 때만 나타나는 **[선택 복사]** 버튼을 통해 나만의 필터 정규식을 즉시 생성할 수 있습니다.
    - **링크 비활성화**: 개별 선택 편의성을 위해 기존의 클릭 시 거래소 이동 기능은 주석 처리되었습니다.
- **정규식 포맷 고도화**: 공백으로 인한 매칭 오류를 방지하기 위해 `^scarab.of.the.c`와 같이 단어 사이에 마침표(`.`)를 사용하도록 개선했습니다.
- **네버싱크(NeverSink) 필터 색상 적용**: 인게임 아이템 필터와 동일한 5단계 티어 배경색 및 텍스트 색상 규칙을 적용하여 직관적인 가치 판별이 가능합니다 (Tier 1~5).
- **PoE 255자 제한 대응 (Chunking)**: 인게임 검색창의 길이 제한을 고려하여, 아이템이 많은 Tier 5(1c 미만)는 35개 단위로 자동 분할하여 여러 개의 버튼(`1c↓ #1`, `1c↓ #2`...)으로 노출합니다.
- **가로 스크롤 버튼 바**: 분할된 버튼이 많아질 경우를 대비해 상단 복사 영역에 가로 스크롤을 적용했습니다.
- **지능형 가격 포맷팅**: 
    - **4c 미만**: 소수점 첫째 자리까지 표시하되 `.0`은 제거하여 정밀도 유지 (예: `0.1`, `1.5`, `2`).
    - **4c 이상**: 반올림하여 정수로 표시하여 시각적 간결함 확보 (예: `4`, `12`, `178`).
- **세련된 복사 버튼**: Material Design 기반의 카드 형태(아이콘, 리플 이펙트 포함)로 리뉴얼하여 조작 피드백을 강화했습니다.
- **다국어 툴팁 및 분류**: 이름뿐만 아니라 분류 정보(Class)도 영문/국문으로 완벽하게 전환됩니다.
- **정규식 복사 최적화**: 영문 모드 사용 시 검색창에 바로 사용 가능한 `"scarab "` 접두어가 포함된 정규식을 제공합니다.
- **렌더링 성능 최적화 (2026-03-18)**:
    - **리스너 통합**: 323개(17x19)의 그리드 셀마다 존재하던 개별 `Obx` 리스너를 제거하고, 상위 수준에서 언어 변경을 한 번에 관찰하도록 수정하여 로딩 속도를 개선했습니다.
    - **연산 최적화**: 리스트 필터링 시 `Set`을 활용한 O(N+M) 알고리즘을 도입하여 메인 스레드 부하를 최소화했습니다.
    - **Controller 기반 데이터 가공**: 뷰 진입 시 발생하는 지연을 줄이기 위해 모든 데이터 정제 로직을 `GetScarabTable` 서비스 계층으로 이전했습니다.

## 3. 잔돈 계산기 (Change Calculator)
- **완전한 i18n 적용**: 모든 입력 필드 레이블, 버튼, 결과 텍스트가 다국어화되었습니다.
- **검증 UI**: 오류 메시지(값 입력 필요, 소수점 제한 등)도 선택된 언어에 맞춰 실시간으로 나타납니다.

## 4. 메인 홈 및 네비게이션
- **메뉴 간소화**: `PageRandomBuildSelector`를 메뉴에서 숨겨 현재 활발히 사용되는 기능 위주로 구성했습니다.
- **반응형 대응**: 세로/가로 모드에 따라 Drawer와 상단 바 구성을 최적화했습니다.

## 5. 날짜 표시 표준화
- 앱 내의 모든 데이터 업데이트 시간은 `yyyy-MM-dd HH:mm` 형식으로 표시됩니다.
